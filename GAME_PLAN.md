# План разработки 3D Шутера

## Обзор проекта

**Цель**: Создать десктопную 3D игру в жанре шутер от первого лица (FPS)

---

## Этап 1: Выбор технологий и настройка проекта

### Варианты технологий:

#### Вариант A: Three.js + Electron (Рекомендуется для быстрого старта)
- **Three.js** — 3D графика в браузере
- **Electron** — упаковка в десктопное приложение
- **Плюсы**: JavaScript/TypeScript, быстрый прототип, кроссплатформенность
- **Минусы**: Производительность ниже чем у нативных движков

#### Вариант B: Unity (C#)
- **Плюсы**: Мощный движок, огромная документация, Asset Store
- **Минусы**: Требует установки Unity Editor, изучение C#

#### Вариант C: Godot (GDScript/C#)
- **Плюсы**: Бесплатный, легкий, открытый исходный код
- **Минусы**: Меньше ресурсов чем у Unity

#### Вариант D: Unreal Engine (C++/Blueprints)
- **Плюсы**: Лучшая графика, Blueprints (визуальное программирование)
- **Минусы**: Сложный для новичков, тяжелый

## Этап 2: Базовая структура игры (на примере Three.js + Electron)

### 2.1 Настройка проекта
```
shooter-game/
├── src/
│   ├── main.ts              # Точка входа Electron
│   ├── game/
│   │   ├── index.ts         # Инициализация игры
│   │   ├── scene.ts         # Настройка сцены Three.js
│   │   ├── camera.ts        # Камера от первого лица
│   │   ├── controls.ts      # Управление (WASD + мышь)
│   │   └── renderer.ts      # Рендеринг
│   ├── entities/
│   │   ├── player.ts        # Класс игрока
│   │   ├── enemy.ts         # Класс врага
│   │   └── weapon.ts        # Класс оружия
│   ├── world/
│   │   ├── level.ts         # Уровень/карта
│   │   └── obstacles.ts     # Препятствия
│   └── ui/
│       ├── hud.ts           # Интерфейс (здоровье, патроны)
│       └── menu.ts          # Меню
├── assets/
│   ├── models/              # 3D модели
│   ├── textures/            # Текстуры
│   └── sounds/              # Звуки
└── package.json
```

### 2.2 Минимальный набор зависимостей
- `three` — 3D графика
- `electron` — десктопное приложение
- `cannon-es` или `ammo.js` — физика
